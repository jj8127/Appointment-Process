# 통합 테스트 체크리스트 (누락 방지 실행본)

## 1) 목적
- 본 문서는 위촉/게시판/설계요청/설정/기타 기능을 누락 없이 검증하기 위한 단일 실행 기준이다.
- 케이스 원본은 `docs/testing/integrated-test-cases.json` 이며, 실행 결과는 `INTEGRATED_TEST_RUN_RESULT.json`으로 관리한다.

## 2) 사용 방법 (필수)
1. 실행 결과 파일 생성
```bash
npm run qa:init:integrated
```
이미 파일이 있으면 `npm run qa:init:integrated -- --force`를 사용한다.
2. 생성된 `docs/testing/INTEGRATED_TEST_RUN_RESULT.json`에서 케이스별 결과 입력
- `status`: `NOT_RUN | PASS | FAIL | BLOCKED | SKIPPED`
- `executedAt`: ISO datetime
- `evidence`: 스크린샷/영상/로그 경로
- `notes`: 실패/차단/스킵 사유
3. 누락 여부 검증
```bash
npm run qa:validate:integrated
```

## 3) 실행 규칙
- 모든 케이스는 독립 실행(케이스별 테스트 데이터 초기화).
- `app + web` 표시는 두 플랫폼 모두 수행.
- `P0`는 릴리즈 차단 게이트로 운영한다.
- PASS 케이스는 증적 필수, FAIL/BLOCKED/SKIPPED는 사유 필수.

## 4) 케이스 카탈로그

### 4.1 위촉
- `ONB-01` 회원가입 위촉 4경우 독립 실행(`none/life_only/nonlife_only/both`)
- `ONB-02` 본등록 후 양트랙 완료 FC만 5단계 완료 표시
- `ONB-03` 총무 화면(앱/웹) 위촉 상태별 FC 표시 정확성
- `ONB-04` FC 수당동의 + 총무 승인/반려
- `ONB-05` 총무 서류요청 + FC 제출서류 승인/반려
- `ONB-06` FC 서류 제출칸 생성/제출 동작
- `ONB-07` 총무 위촉차수 입력/승인/반려(모든 경우의 수)
- `ONB-08` FC 위촉차수 표시 + 위촉완료일 입력(모든 경우의 수)
- `ONB-09` 수당동의 전 시험접수 차단 + 완료 후 신청/수정/취소
- `ONB-10` 총무 시험 등록 처리(FC 신청 건)

### 4.2 게시판
- `BRD-01` 본부장/총무 게시글 등록
- `BRD-02` 본부장 본인 글 수정/삭제
- `BRD-03` 파일첨부 + 이미지 미리보기
- `BRD-04` 첨부 이미지 순서 변경
- `BRD-05` 게시글 유형(카테고리) 노출 정확성
- `BRD-06` FC/총무/본부장 감정표현 추가/취소/변경
- `BRD-07` 총무 모든 글 등록/삭제 권한
- `BRD-08` 댓글/대댓글/좋아요/수정/삭제
- `BRD-09` 공지 분류 글의 위촉 페이지 미리보기 노출

### 4.3 설계요청 (앱/웹 동시 검증)
- `RB-01` 통계(전체/대기/진행/완료) 앱-웹 동시 반영
- `RB-02` 주소 복사/사용법 버튼 동작
- `RB-03` 설계매니저 완료 의뢰 승인/거절 처리
- `RB-04` 메신저 파일 송수신
- `RB-05` 설계매니저 거절/수락/완료/완료첨부파일
- `RB-06` 설계매니저 하단네비 2탭(설계요청/설정) 고정
- `RB-07` 설계 알림 수신(설계매니저/FC/본부장/총무)
- `RB-08` FC/본부장 설계코드 추가/수정/삭제

### 4.4 설정
- `SET-01` FC/본부장/총무 본인 계정 삭제 완전성
- `SET-02` 총무의 FC 삭제 완전성(앱/웹)

### 4.5 기타
- `ETC-01` FC 메신저 본부장/총무 목록 노출
- `ETC-02` 본부장/총무 메신저 FC 목록 노출
- `ETC-03` 메신저 파일 첨부 + 메시지 전송
- `ETC-04` 읽음/안읽음 알림 구분 동작

### 4.6 추가 필수 (P0)
- `P0-01` 계정 전환 시 권한/하단네비/데이터 캐시 격리
- `P0-02` 본부장(readOnly) 권한 우회 차단(URL/API)
- `P0-03` 상태 전이 무결성(비정상 순서 차단)
- `P0-04` 동시 작업 충돌 일관성(총무 앱/웹)
- `P0-05` 버튼 연타 시 중복 처리 방지
- `P0-06` 네트워크 실패 복구/재시도/에러 메시지
- `P0-07` 푸시 클릭 딥링크 정확성
- `P0-08` 삭제 후 고아 데이터 부재(DB/Storage/참조)
- `P0-09` 업로드 경계값(대용량/오류확장자/0바이트/취소/재시도)
- `P0-10` 마감 시점 경계(자정/타임존) 처리
- `P0-11` 앱 회원가입/비번설정 -> request_board 웹 로그인 연동
- `P0-12` 앱 비밀번호 변경/재설정 -> request_board 웹 동기화
- `P0-13` 비밀번호 동기화 실패 복구(재시도/다음 로그인)

### 4.7 추가 권장 (P1)
- `P1-01` request_board 브릿지 토큰 만료/무효 처리
- `P1-02` 다기기 동시접속 동기화(읽음/메시지/진행상태)
- `P1-03` 대량 데이터 성능/누락 검증
- `P1-04` 구버전 앱 호환성(마이그레이션 이후)
- `P1-05` 비관리자 경로 개인정보 노출 점검
- `P1-06` 장애 관찰 가능성(로그/모니터링 추적성)

## 5) 증적 기준
- 최소 1개 이상: 화면 캡처 또는 녹화 + 서버 로그/DB 쿼리 결과.
- 상태 전이 케이스는 "변경 전/후" 증적을 모두 남긴다.
- 삭제/동기화 케이스는 DB 확인 쿼리 결과를 notes 또는 evidence에 링크한다.
