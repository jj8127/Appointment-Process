# 추천인 테스트 데이터 운영 규칙

## 1. 목적

- 추천인 시스템 테스트에 쓰는 계정, 기기, 링크, 초기화 규칙을 한 곳에 모은다.
- 구현 후 실제 테스트용 전화번호/코드가 정해지면 이 파일을 먼저 갱신한다.

## 2. 기본 원칙

- 운영 실사용 계정을 테스트에 재사용하지 않는다.
- 동일 케이스 반복 시 계정 상태가 누적되면 notes에 남기지 말고 여기의 초기화 절차를 먼저 갱신한다.
- 테스트 계정은 역할과 목적이 이름만 봐도 드러나게 유지한다.

## 5. 초기화 체크리스트

구현 후 아래 항목을 실제 스크립트/명령으로 채운다.

1. invitee 계정 삭제 또는 추천 관계 초기화
2. pending attribution 삭제
3. 테스트용 이벤트 로그 범위 정리
4. 앱 로컬 저장소/딥링크 캐시 초기화
5. 스토어 referrer 또는 브라우저 세션 상태 초기화

## 6. 유지 규칙

- 실제 전화번호를 채웠으면 이 파일을 최신으로 유지한다.
- 새 음성/문자/스토어 제약이 생기면 notes가 아니라 이 파일에 추가한다.
